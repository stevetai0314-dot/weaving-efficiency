織布效率記錄 系統流程圖

weaving-efficiency · GitHub Pages + Google Apps Script + Google Sheets · 398 台機器

主流程

graph TD A([開啟頁面]) --> B[選擇工廠\n織一 / 織二 / 織三] B --> C[從 GAS 載入\n前一班機時數] C --> D[顯示 Zone 分頁] D --> E[Zone 內機台\n依走動路線陣列順序排列] E --> F[輸入當前機時] F --> G[即時計算效率\n當前 - 前班 ÷ 120 × 100%] G --> H{效率值} H -- ≥ 85% --> I[綠色顯示] H -- 60–84% --> J[橘色顯示] H -- < 60% --> K[紅色顯示] I & J & K --> L{全部輸入?} L -- 否 --> F L -- 是 --> M[批次送出] M --> N[(Data 工作表\n附時間戳記)] style I fill:#e6f4ea,stroke:#1e8e3e style J fill:#fce8b2,stroke:#e65100 style K fill:#fce8e8,stroke:#d32f2f style N fill:#e6f4ea,stroke:#1e8e3e

走動路線邏輯（最重要的設計）

graph LR A[ZONES 陣列定義] --> B[factory: 織一\nlabel: 第一區\nmachines: 1,2,179,4,181...] A --> C[factory: 織三\npfx: T3_\nmachines: 68,67,66,86...] B --> D[渲染時依陣列順序\n不做任何 sort] C --> D D --> E[機台 1,2,179,4,181\n按這個順序顯示在畫面上] E --> F[工人按此順序走動\n不需要回頭找機台] style D fill:#fce8b2,stroke:#e65100 style F fill:#e6f4ea,stroke:#1e8e3e

跨廠機台 ID 衝突解法

graph TD A[織一 機台 1] --> B[key: 1] C[織三 機台 1] --> D[key: T3_1\npfx: T3_] B & D --> E[DOM 和資料都用 key\n避免覆蓋] E --> F[payload 附 factory 欄位\nSheets 依此區分] style D fill:#e8f0fe,stroke:#1a73e8

重點說明

  • 走動路線 ZONES 陣列的機台順序就是工人的實際走動路線，數字不連續是正常的（機台安裝順序 ≠ 擺放順序）
  • 效率計算 公式：(當前機時 - 前班機時) ÷ 120 × 100%，120 分鐘為一班
  • 前班資料 頁面開啟時從 GAS GET 載入，跳過已停機的機台不帶入基準值
  • 398 台機器橫跨織一/織二/織三，機台 ID 在不同廠有重疊，用 prefix 解決
  • 中越雙語切換，語言選擇存在 localStorage
