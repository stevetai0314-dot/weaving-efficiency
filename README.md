# 織帶機台效率登記系統

**🔗 使用連結：[https://stevetai0314-dot.github.io/weaving-efficiency/](https://stevetai0314-dot.github.io/weaving-efficiency/)**

手機版表單，讓各廠班別人員逐台輸入時數，自動計算稼動率並存入 Google Sheets。

---

## 功能概覽

- **廠別切換**：織一 / 織二 / 織三，分區獨立記錄
- **分區標籤頁**：織一 4 個區、織二 2 個區、織三 1 個區，共管理 **398 台機台**
- **自動稼動率**：輸入時數後即時顯示（綠 ≥85%、橘 60–84%、紅 <60%）
- **停機標記**：一鍵標記停機，不計入稼動率
- **上次時數自動帶入**：開啟即從 Google Sheets 讀取每台機台上次登記值
- **雙語介面**：中文 / 越南文 切換
- **送出確認彈窗**：顯示開機 / 停機 / 未填 台數及整體稼動率

## 部署流程

### 1. Google Sheets 設定

使用此試算表作為資料庫：
[試算表連結](https://docs.google.com/spreadsheets/d/1cVAV8odDf457CJHGuV_Yy7JOB97vbiCWy2Y4x-bfE4M)

系統會自動建立「登記紀錄」工作表，表頭如下：

| 時間戳記 | 廠別 | 班別 | 組別 | 機台號 | 上次時數 | 本次時數 | 稼動率% | 停機 |

### 2. Google Apps Script 部署

1. 開啟試算表 → **擴充功能 → Apps Script**
2. 將 `Code.gs` 內容貼上取代預設程式碼
3. 點選 **部署 → 新增部署作業**
4. 類型選「**網路應用程式**」
5. 執行身分：**我自己**；存取權：**所有人**
6. 複製部署後的 **網路應用程式 URL**

### 3. 填入 API URL

在 `index.html` 第 233 行找到：

```javascript
const GAS_URL = 'https://script.google.com/macros/s/...';
```

換成步驟 2 複製的 URL。

### 4. 發布前端

將 `index.html` 部署至任何靜態網站服務即可，例如：

- **GitHub Pages**（本 repo 直接啟用）
- Google Drive 分享
- 任何靜態主機

## 使用說明

1. 開啟網頁，選擇**廠別**（織一 / 織二 / 織三）
2. 選擇**班別**（白班 / 夜班）及**組別**（A / B / C / D）
3. 逐台輸入本次時數（**不需打小數點**，例：138.4 小時 → 輸入 `1384`）
4. 停機台數按**停機**按鈕標紅
5. 全區填完後按**確認送出**，確認摘要後送出

---

## 稼動率計算方式

```
稼動率 = (本次時數 − 上次時數) ÷ 120 × 100%
```

> 120 = 去除小數點後的換算係數（÷10）× 每班 12 小時

## Code.gs API 說明

| 方法 | 用途 |
|------|------|
| `GET` | 讀取每台機台最新一筆非停機時數，回傳 `{ "廠別:機台號": 時數 }` |
| `POST` | 接收前端送出的班別資料，寫入試算表 |

POST 請求格式：

```json
{
  "factory": "織一",
  "shift": "day",
  "group": "A",
  "records": [
    { "machineId": 1, "lastHours": 13840, "currentHours": 13960, "efficiency": 100.0, "stopped": false }
  ]
}
```

## 注意事項

- 修改 `Code.gs` 後需**重新部署**（新增部署版本），舊 URL 不會自動更新
- 織三廠的機台號（1–86）與織一重疊，系統以廠別欄位區分，不會混淆
- 舊資料（無廠別欄位）系統自動視為「織一」，向下相容
